动作编辑器¶

动作编辑器 是你可以定义和控制 动作 的地方。它使你能够查看和编辑在 函数曲线 编辑器中定义为动作的函数曲线数据块。因此它是介于低级 函数曲线 和高级 NLA编辑器 之间。

它为您提供了函数曲线数据块的略微简化视图(有点类似于没有控制柄的函数曲线)。编辑器可以同时列出物体的所有动作数据块。

每个动作数据块都形成一个顶级通道 (见下图)。请注意，一个物体可以有几个 约束 (每个动画化约束对应一个) 和 姿态 (对于骨架，每个动画化骨骼对应一个) 函数曲线数据块，因此一个动作可以有这些通道中的几个。

标题栏¶

上一层/下一层(向下/向上箭头图标)

在NLA堆栈中彼此堆叠/暂存的不同动作之间切换，无需转到NLA编辑器和离开调整模式并在其他片段上重新输入。

单击向上/向下箭头按钮，将转到NLA片段(其中的动作正在动作编辑器中进行调整)上方/下方的NLA轨道中的动作。

如果同一图层/轨道上有多个动作/片段，则只使用最接近当前帧的动作/片段。

操作将单独考虑查看/编辑动作的设置(如Solo和NLA Muting )。这样做是为了更容易预览不同的暂存动作。

  • 如果从一个单独(solo)的NLA轨道移动到活动动作，则NLA轨道将被静默(不可用)，以便可以单独编辑动作。

  • 同样，如果NLA堆栈在编辑动作时被静默，堆栈下面的NLA轨道将在启用solo的情况下被编辑。

  • 如果在NLA轨道之间切换，则前一轨道的单独状态将被传送到新轨道。

Note

当您不使用NLA片段编辑动作时，这些仍然有效。 如果您只是正常动画化一个新动作，可以使用“向下箭头”暂时跳转到上一个动作而不会丢失您正在处理的新动作，然后一旦你完成了检查其他动作，使用“向上箭头”回到它。

动作

数据块菜单 。

添加 +

当一个动作被创建时，它被存储在NLA动作暂存器中。

取消链接 X

当 Shift-LMB 点击它清除伪用户并且从NLA堆栈中移除暂存的动作。

下推(双向下箭头图标)

将活动动作添加到NLA堆栈作为作用长片段。这与在NLA编辑器中按下下推按钮基本相同。

存放(雪花图标)

将活动动作暂存在NLA堆栈上。即它以与创建新动作相同的方式添加为非贡献(起作用)堆栈。

Note

在这两种情况下(下推和存放)，一旦该动作被添加到NLA堆栈中，它就从活动动作槽中清除/取消分配(即不能从动作编辑器/曲线编辑器中编辑，除非您稍后在相应片段上进入的“调整模式”)。
